# 문제 정의

## 1. 현재 프로세스의 한계

### 공통 (수기 육안검사)

| 문제 | 설명 |
|---|---|
| 재현성 부족 | 검사자의 숙련도·피로도·조명 환경에 따라 판정이 달라짐 |
| 속도 한계 | 전수검사가 어려워 샘플링 검사에 의존, 누락 위험 존재 |
| 기록 부재 | 어떤 위치에서 어떤 불량이 발생했는지 정량 기록이 남지 않아 재발 방지·원인 분석이 어려움 |
| 미세 결함 취약 | 점·기호 누락, 자간 변형, 얇은 칼날 마모 등은 육안으로 놓치기 쉬움 |

### 과제① 인쇄판 문안검사 — 구체적 실패 사례

- 오탈자: "토너" → "토노"
- 문자 누락: "나이아신아마이드"에서 '아' 누락
- 점 누락: 문장 끝 마침표 누락
- 번짐: 잉크 확산으로 글자가 뭉개져 판독 불가

→ 특히 **성분표시 누락·오탈자**는 화장품·식품 등 규제 대상 라벨에서 **법적 리스크**로 직결된다. 사람이 매 LOT 전수로 원본과 인쇄물을 한 글자씩 대조하는 것은 시간·인력상 비현실적이다.

### 과제② 목형 칼날검사 — 구체적 실패 사례

- 휨: 칼날이 설계 기준선보다 바깥으로 굽음
- 끊김: 칼날 선이 중간에 끊어짐 (절단 불량)
- 위치 오차: 모서리·기준점이 설계 위치에서 밀려남
- 마모: 사용 누적으로 칼날 에지가 흐려지고 약해짐 (연속적으로 진행되는 양)

→ 불량 목형이 생산에 투입되면 **다이커팅(톰슨) 불량 → 전체 LOT 재작업/폐기**로 이어져 손실이 크다. 반면 목형 실물 검사는 사람이 형상을 자로 재거나 눈으로 대조해야 해 정밀도와 속도 모두 부족하다.

## 2. 데이터 관점의 근본 난제

| 구분 | 과제① 인쇄판 | 과제② 목형 |
|---|---|---|
| 핵심 난제 | 불량 샘플 희소 / 원본-인쇄물 정합쌍 필요 / 미세 결함(점·자간) | 불량 실물 확보 극히 어려움 / 형상 다품종 / 마모는 연속량(이진 분류 부적합) |
| 왜 어려운가 | 정상 인쇄물은 넘치지만, 법적 리스크가 큰 오탈자·성분표시 오류의 실제 샘플은 거의 존재하지 않음 (애초에 불량은 출하 전 걸러짐) | 정상 목형만 다량 존재. 실제 파손·마모 목형은 폐기되어 사라지므로 학습용 실불량 수집이 근본적으로 제한적 |

즉, **"정답과 실물을 비교해 다른 곳을 찾는다"는 원리는 단순하지만, 학습에 쓸 불량 데이터가 태생적으로 부족**하다는 것이 두 과제의 공통 핵심 문제다. 이 문제를 어떻게 우회할지가 프로젝트 성공의 관건이며, 그 해법(합성 결함 증강, 이상탐지 접근)은 `spec.md`에서 다룬다.

## 3. 이 프로젝트가 해결해야 하는 것

1. 원본/도면과 실물을 자동으로 정밀 정렬하고 차이를 자동 검출한다 (사람의 대조 작업 대체)
2. 불량 데이터 희소성을 합성 증강·이상탐지 방식으로 우회하여 실사용 가능한 모델을 만든다
3. 치명 결함(성분표시 오류, 칼날 끊김·휨)에 대해 **미검출 0**을 지향하는 검사 신뢰도를 확보한다
4. 검사 결과(불량 유형·위치·좌표)를 구조화된 데이터로 남겨 MES와 연계, 추적 가능성을 확보한다
