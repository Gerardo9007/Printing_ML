# 기술 명세

## 1. 시스템 개요

두 과제 공통 파이프라인:

```
[촬영/입력] → [전처리·정규화] → [정합(Registration)] → [차분/이상탐지] → [판정·시각화] → [결과 기록(MES 연계)]
```

| 항목 | 과제① 인쇄판 문안검사 | 과제② 목형 칼날검사 |
|---|---|---|
| 입력 | 인쇄판/1차 인쇄물 이미지, 디자인 원본(AI/PDF) | 목형 이미지, 설계 CAD/도면 |
| 핵심 기술 | 이미지 정합 + OCR + 픽셀 차분 | 형상 정합 + 이상탐지(Anomaly Detection) + 결함 세그멘테이션 |
| 출력 | 불량 종류 + 좌표(BBox/영역) + 텍스트 diff | 불량 종류 + 영역 세그멘테이션 + 위치오차(mm) |

## 2. 검출 클래스 정의

### 과제① 인쇄판 문안검사
- 정상
- 문자누락
- 오탈자
- 점누락 (마침표·기호)
- 번짐
- 성분표시오류 (누락/위치오류) — **최우선 치명 클래스**

### 과제② 목형 칼날검사
- 정상
- 휨
- 끊김 (절단 불량) — **최우선 치명 클래스**
- 위치오차
- 마모 (연속량 → 등급화: 정상/주의/교체, 또는 회귀)

## 3. 데이터 전략

### 3.1 수집

| 단계 | 과제① | 과제② |
|---|---|---|
| 정상 데이터 | 승인 완료 인쇄판/1차 출력물, 600dpi 이상 스캔, 제품·폰트·언어별 폭넓게 확보 | 정상 목형을 형상·제품·크기별로 다량 촬영 |
| 정답(GT) | 디자인 원본을 동일 배율로 라스터화 | 설계 CAD/도면을 기준 형상으로 등록 |
| 정합쌍 | 동일 LOT (원본 ↔ 인쇄물) 1:1 매칭 | 목형 ↔ CAD 오버레이 매칭 |
| 실불량(NG) | 검사 반려품을 유형·좌표와 함께 아카이빙 (test셋 중심) | 폐기 목형 재활용 + 인위적 손상 샘플 제작 (극희소, test셋 중심) |
| 촬영 표준화 | 조명·배경·해상도·정렬·ICC 색프로파일 고정, 스캐너/라인스캔카메라 지그 | 고정 지그·동일 각/거리, 백라이트(실루엣) 또는 텔레센트릭 렌즈로 에지 캡처, 반사 통제 |
| 메타데이터 | 제품코드·LOT·인쇄판버전·원본링크·검사결과·불량유형·좌표 (MES 연계) | 목형ID·제품·검사결과·불량유형·불량위치·누적 사용횟수/타발수량 (MES 연계) |

### 3.2 증강 (불량 희소성 우회 — 핵심 전략)

원본/기준 형상에서 결함을 프로그램으로 생성하여 라벨을 자동 부여하는 **합성 결함 생성**이 두 과제 공통 최우선 기법이다.

| 기법 | 과제① | 과제② | 비고 |
|---|---|---|---|
| 합성 결함 생성 (문자 삭제/치환/이동, 점·기호 삭제) | ○ | — | 최대 효과 기법 |
| 합성 결함 삽입 (국소 끊김·휨·마모·위치이동) | — | ○ | 최대 효과 기법 |
| 번짐/잉크확산 시뮬레이션 | ○ | △(마모 표현) | 가우시안 블러, 모폴로지 팽창 |
| 기하 회전 | △ (±1~2°, 과회전·반전 금지) | ○ (0~360°) | ①은 텍스트 판독 훼손 주의 |
| 좌우/상하 반전 | ✕ (문자 의미 파괴) | △ (형상 대칭 시만) | |
| 노이즈/밝기·대비 지터 | ○ | ○ | 촬영·조명 편차 대응 |
| CutPaste/CutMix | △ | ○ | ②이상탐지 보강에 특히 효과적 |
| Copy-Paste (실불량 패치 이식) | ○ | ○ | 소량 실불량을 대량으로 확장 |

### 3.3 보정·정제

- **정합(Registration)**: 특징점·호모그래피(①) / CAD 오버레이·스케일 캘리브레이션(②)로 정밀 정렬 후 차분 — 정렬 오차로 인한 오검출 제거의 핵심
- **전처리 정규화**: 데스큐, 배경 제거, 해상도/명암/색상 정규화, (②)에지 추출(Canny)·편광으로 반사 억제
- **라벨 검수**: 이중검수(2인 교차검수), 불일치 조정 룰, 좌표 정합 확인
- **클래스 불균형 보정**: 합성:실측 비율 조정, focal loss, 가중 샘플링
- **데이터 분할**: 제품·LOT 단위(①) / 목형ID·제품 단위(②)로 train/val/test 분리(누수 방지). 실불량은 test에 충분히 배치. 예시 비율 train/val/test = 70/15/15

## 4. 모델링 접근

- **과제①**: 참조기반 이상검출 = Registration + OCR + Diff. 문자 인식은 OCR 엔진 + 혼동쌍 사전(O/0, l/1, ㅇ/0) 보정
- **과제②**: 정상 데이터 다량 기반 이상탐지(PatchCore, PaDiM 등) 우선 적용. 불량 라벨 부족을 이진분류 대신 이상탐지로 대응. 마모는 회귀 또는 등급 라벨링(정상/주의/교체)으로 이진 분류 한계 보완

## 5. 라벨링 스키마

| 항목 | 과제① | 과제② |
|---|---|---|
| 라벨 형식 | 불량 종류 + BBox/영역 좌표 + 텍스트 diff | 불량 종류 + 세그멘테이션/좌표 + 위치오차(mm) |
| 라벨 도구 | OCR 자동 diff + 사람 검수 (예: Label Studio) | 세그멘테이션 툴 + 기준형상 오버레이 |
| 검수 방식 | 2인 교차검수 + 불일치 조정 | 2인 교차검수 |

## 6. 목표 성능 지표

| 항목 | 과제① | 과제② |
|---|---|---|
| 최우선 지표 | 재현율(치명 클래스 미검출 = 0 목표) | 결함 재현율(끊김·휨 미검출 = 0 목표) |
| 보조 지표 | Precision, F1, mAP | IoU, mAP, 위치오차(mm) |
| 운영 원칙 | 오검출(과검출)은 재검 프로세스로 흡수, 미검출을 절대 우선 최소화 | 동일 |
| 목표 성숙도 단계 | 사업계획 기준 "기초 → 중간1" 수준 | 동일 |

## 7. 비기능 요구사항

- **추적성(Traceability)**: 모든 검사 결과는 제품코드·LOT/목형ID와 연계되어 MES에서 조회 가능해야 함
- **촬영 표준화**: 지그·조명·해상도가 고정되지 않으면 모델 성능이 보장되지 않음 — 촬영 환경 표준화가 데이터 수집 이전에 선행되어야 함
- **버전 관리**: 디자인 원본(①)·CAD 도면(②)의 버전이 실제 생산 버전과 일치하는지 확인하는 절차 필요 (버전 불일치는 오검출의 주요 원인)
- **설명 가능성**: 검사자가 결과를 신뢰하고 재검 여부를 판단할 수 있도록, 판정 근거(불일치 위치 시각화)를 반드시 함께 제공

## 8. 미해결·후속 검토 사항

- 실불량(NG) 샘플의 목표 최소 수량 및 확보 일정 (test셋 신뢰도 확보 기준)
- 엣지 디바이스 vs 서버 추론 여부 (라인 속도·지연 요구사항에 따라 결정)
- 합성 데이터 비율이 실제 분포와 얼마나 괴리되는지에 대한 정기 검증(리얼-합성 갭 모니터링) 절차
