## 0. 개요 — 무엇을 만들었고 무엇을 하지 않았는가

### 0.1 프로토타입의 범위와 한계 (반드시 읽을 것)

**핵심 메시지**: 이번 완성물은 "**전체 목형검사 시스템이 아니라, 정합+diff 원리 검증 프로토타입**"이다.

#### 만든 것
- **정합(Registration) + 차분(diff) 기반 결함검출 파이프라인**: CAD 도면과 실물 목형 칼날을 겹쳐 오버레이한 뒤, 기하학적 차이로 휨·끊김 결함을 검출하는 고전적 접근법의 **원리만** 합성 데이터로 검증
- **치명 클래스(끊김·휨) 2종 검출**: spec §2가 요구하는 5클래스(정상/휨/끊김/위치오차/마모) 중 **최우선 2종만** 프로토타입 범위로 한정
- **단일·다중 결함 시나리오 6건(case0~6)**: 정상, 휨만, 끊김만, 복합, **다중 동일결함(QA 지적 회귀검증 포함)**

#### 만들지 않은 것 (범위 외 — 이번 사이클 자동 개선 범위 밖)
- ❌ 실제 목형 촬영 이미지·CAD 도면 데이터 (완전 합성)
- ❌ 이상탐지 모델(PatchCore/PaDiM) — spec §4 지정 but 고전적 diff 방식만
- ❌ 마모 클래스(연속량 등급화)
- ❌ 위치오차 독립 클래스
- ❌ 검사자 UI 5단계(화면①~⑤) — 설계문서만 존재
- ❌ 도면 버전 게이트, 촬영 표준화 게이트
- ❌ HITL 강제 재검 상태머신
- ❌ MES 추적성/이력 대시보드
- ❌ 완전 무인 자동판정 (Non-goal, 항상 사람 재검 전제)

**이것이 의미하는 바**: 합성 데이터 위에서 "정합과 차분이 원리적으로 동작한다"는 것만 보였으며, 실사용 성능·일반화 가능성·모델 접근 최적성은 **별도로 실측 데이터 확보 후 재검증 필요**하다. 이번 보고서의 "성과"는 프로토타입 범위 내 완성도이지, 전체 시스템 완성이 아니다.

### 1.3 Do 단계 (2026년 4월 ~ 6월)

**문서**: `docs/dev/task2-die-blade/README.md` (구현 문서) + `die_blade_qc_demo.py` (코드)

#### 구현 내용
1. **기준 도면 생성**: 합성 사각 다이라인 (폐곡선, 둥근 모서리)
2. **결함 주입**:
   - 휨(`inject_bend`): Hann 창으로 부드러운 변위 (바깥쪽 법선 방향, 최대 6.8mm)
   - 끊김(`inject_break`): 점 제거로 실제 단절 (길이 8~24mm)
3. **촬영 정렬오차 시뮬레이션**: ±1.4° 회전, ±수px 이동
4. **정합(Registration)**: OpenCV `findTransformECC`(MOTION_EUCLIDEAN) — 두께 팽창(dilate) 마스크로 회전+이동 추정
5. **차분(diff) 검출**: `cv2.distanceTransform` 양방향 거리장 — 끊김(도면은 있으나 실물 없음), 휨(실물이 도면 밖으로 나감) 구분
6. **후처리**:
   - 연결요소 분석: bbox, centroid, area → 결함 단위 묶음
   - 최소 길이 필터: `MIN_DEFECT_ARC_MM = 2.0mm` (노이즈 억제)
   - 휨/끊김 중복 해소: bbox 겹치는 경우 `"복합(휨+끊김 의심)"` 태깅 (G5 대응)
7. **시각화·기록**:
   - PNG: 기준도면(회색) + 정합실물(녹색) + 결함 bbox(빨강/주황) + mm 수치
   - JSON: 정합 변환행렬, 주입/검출 결함 리스트, 임계치, 신뢰도

### 1.5 Act 단계 (2026년 7월, PDCA 자동 개선 1회차)

**실행 범위**: 코드 자동수정 가능한 항목만 집중 (G1·G2·G6~G11은 범위 밖 명시)

#### 대응 완료 항목

##### 1) QA §2-C — 최소 결함 길이 필터 근거 실측 (신규 기능)

**문제**: `MIN_DEFECT_ARC_MM = 2.0mm`가 "과검출 억제" 목적이라고만 했지, 실제 근거 미확인.

**대응 방법**: 
- 신규 함수 `sweep_min_defect_arc_threshold(threshold_candidates_mm)` 추가
- 임계값 0.5mm~10.0mm에서 정상 케이스 과검출·길이별 결함 미검출 실측
- 결과 정합 및 해석을 README에 기록

**실측 결과** (실제 실행값):
```
임계값(mm) | 정상 과검출 | 0.1/0.3/0.5/1.0/1.5/2.0/2.5/3.0/4.0 mm 미검출 여부
-----------|-----------|--------------------------------------------------
0.5~3.0    | 0건        | X, X, X, X, O, O, O, O, O  (거의 동일)
5.0        | 0건        | X, X, X, X, X, X, X, X, O  (필터 시작)
10.0       | 0건        | X, X, X, X, X, X, X, X, X  (전부 필터)
```

**발견(중요)**: **"MIN_DEFECT_ARC_MM 값 자체가 검출 결과를 결정하지 않는다"**
- 0.5mm~3.0mm 범위 내에선 임계값이 거의 영향 없음
- 진짜 결정 요인: **`DEFECT_TOL_MM(허용오차 1.0mm)` + 라인두께/팽창커널 기하학** = 1.26mm~1.89mm 절벽
- `MIN_DEFECT_ARC_MM`이 실제로 결함 필터하기 시작: 4~5mm 이상
- **현재값 2.0mm는 이 위험 구간(4~5mm)과 충분한 안전거리** ✅

**의의**: 값 변경은 하지 않되(2.0mm 유지), 데이터-기반 근거를 README에 확보. 향후 실데이터(노이즈 포함) 확보 후 "과검출 억제" 측면은 재검증 필요.

##### 2) Gap G5 — 휨/끊김 분류 정교화 (신규 후처리)

**문제**: 휨 결함 발생 시 같은 위치가 부수적으로 "끊김"으로도 검출되는 경향(미검출0 목표에는 유리하나, 유형 통계·재검 우선순위 혼선 초래).

**대응 방법**:
- 신규 함수 `resolve_bend_break_overlap(defects)` 추가
- **bbox가 겹치는 경우만** 편차가 큰 쪽을 `"복합(휨+끊김 의심)"`으로 명시 재태깅
- 별개 휨·끊김(다른 위치)은 손대지 않음
- 색상: 자홍색으로 시각화, JSON/PNG/콘솔에 노출

**회귀 검증**:
```
case0_normal:           주입=없음 검출=없음 → PASS
case1_bend_only:        주입=[휨] 검출=[복합] → PASS
case2_break_only:       주입=[끊김] 검출=[끊김] → PASS
case3_bend_and_break:   주입=[끊김, 휨] 검출=[끊김, 복합] → PASS
case4_multi_break:      주입=[끊김3] 검출=[끊김3] (다중검증) → PASS
case5_multi_break_and_bend: 주입=[끊김2, 휨] 검출=[끊김2, 복합] → PASS
case6_multi_bend:       주입=[휨2] 검출=[복합2] (다중검증) → PASS
```
**전체 PASS** ✅

## 2. 핵심 성과 (Achievements)

### 2.1 QA 지적 치명 버그(§2-A) 근본원인 수정 + 회귀검증 완료

**원본 문제**: 끊김을 3개 동시에 주입하면 1개만 검출 (**66% 미검출** — spec 6절 목표 정면 위배)

**근본 원인 분석** (2가지):
1. **폐곡선 랩어라운드 병합 오병합**: 
   - `inject_break()`에 "폐곡선을 한 번 자르면 열린 호가 되고, 처음/끝 점이 만나면 병합"이라는 로직
   - 문제: **두 번째·세 번째 끊김이 이미 열린 호(첫 컷의 결과)에 적용될 때도 조건 우연히 참** → 분리되어야 할 두 조각을 다시 이어붙임(bridging) → 시각적으로 사라짐

2. **`start_frac` 기준점 드리프트**: 
   - 두 번째·세 번째 컷이 "원본 도면 둘레 기준 위치"가 아니라 "그 시점 남은 조각의 로컬 위치"로 해석 → 실제 주입 위치가 의도와 상이

**수정 방법** (근본 대응):
- `Arc(points, orig_idx, is_closed)` 구조 도입:
  - `orig_idx`: 각 점이 원본 폐곡선에서 가졌던 인덱스 (조각이 잘려도 유지)
  - `is_closed`: 이 조각이 "아직 컷이 없는 원본 폐곡선 자체"인지 여부
  - 랩어라운드 병합은 `is_closed=True`일 때(최초 컷)만 수행 → 이후 `False`로 고정
- `start_frac`을 항상 `n_total`(=원본 전체 폐곡선 점 수)를 기준으로 환산 → 일관성 확보

**회귀 검증 결과**:
| 항목 | 수정 전 | 수정 후 |
|---|---|---|
| polyline_count (분리된 호 개수) | 1 | **3** |
| 연결요소 개수 | 1 | **3** |
| 검출된 끊김 개수 | 1 / 3 (67% 미검출) | **3 / 3 (0% 미검출)** |

- case4(끊김 3개) 신규 회귀테스트 추가 → 3/3 검출 확인 ✅
- 기존 case0~3 재실행 → 수정 전 값과 완전히 동일(회귀 없음) ✅

**이 수정의 의미**:
- spec §6 핵심 목표 "**끊김 미검출 0**"에 대한 **신뢰도 대폭 상승**
- 다중 파손이 가능한 실제 목형에도 적용 가능한 상태로 전환
- **이것만으로도 이번 사이클의 가장 큰 성과**

### 2.2 ECC 정합 신뢰도 가드레일 추가 (§2-B)

**문제**: 회전 오차 45°~90°에서 `findTransformECC`가 180° 대칭으로 잘못 수렴하면서도 `converged=True` 반환 → 사람이 정합 실패를 감지 불가능

**수정**: 잔차 기반 독립 검증 필드 추가
- `compute_registration_residual()`: 정합된 두 마스크 간 **대칭 평균 최근접거리**(chamfer distance) 계산
- 임계치 `10px = 2.0mm`로 설정 (실측 근거):
  - 정상 정합(1.4°~35°): 잔차 ~1.3px (안전)
  - 대칭 오수렴(45°~90°): 잔차 49px 이상 (위험)
  - **40배 이상 차이** = 명확한 간극

- 결과: `registration_reliable = registration_converged AND residual <= threshold`
- `reliable=False`일 때도 검출 억제 없이 신뢰불가 상태만 명시 노출 (사람 재검 유도)

**회귀 검증**:
```
case0~6 모두: registration_reliable = True, 잔차 0.03~0.48mm (충분히 낮음)
기존 검출 수치: 모두 이전과 동일 → 회귀 없음 ✅
```

### 2.3 최소 결함 길이 필터 실측 근거 확보 (§2-C)

위의 1.5절 Act 단계 상세 참고.

**핵심 발견**: `MIN_DEFECT_ARC_MM` 자체보다 **`DEFECT_TOL_MM`(허용오차)과 라인두께/팽창커널 기하학**이 실제 검출 성능을 결정 → 향후 이 파라미터들이 재조정 대상임을 명확히

### 2.4 휨/끊김 분류 정교화 (G5)

위의 1.5절 Act 단계 상세 참고. **같은 위치 중복 검출 명시화** → 통계·재검 우선순위 혼선 해소 ✅
