## 0. 요약 (Executive Summary)

과제① 프로토타입과 동일하게, 현재 구현물은 **전체 시스템이 아니라 `spec.md` 공통 파이프라인 중 "형상 정합(Registration) + diff 기반 결함검출" 원리만 검증하는 프로토타입**이다(README 스스로 범위를 한정). 검출 대상도 최우선 치명 클래스인 **휨·끊김 2종으로 축소**했고, spec §2가 요구하는 나머지 2종(위치오차·마모)은 명시적으로 범위에서 제외했다. 따라서 두 기준으로 나눠 보고한다.

| 기준 | Match Rate | 해석 |
|---|---|---|
| **전체 설계 범위** (검출 5클래스 + UI 화면 ①~⑤ + 이상탐지 모델 + MES/HITL) | **약 23%** | 검출 2/5클래스, UI·HITL·MES·버전게이트 전부 미구현(의도된 범위 축소). 이번 사이클(QA §2-C, G5)은 축 자체를 늘리진 않고 기존 축의 완성도를 보강 |
| **프로토타입 선언 범위** (휨·끊김 정합+diff 원리검증) | **약 93%** | 원리 검증 달성 + QA 지적 3건(§2-A 다중결함 미검출, §2-B ECC 대각도 오수렴, §2-C 최소길이 필터 근거)을 코드 수정으로 대응 + Gap G5(휨/끊김 분류 정교화) 대응 완료. 단 실데이터(G1)·§3 실데이터 테스트는 여전히 미검증 |

가장 중요한 결론: **과제①과 달리, 치명 클래스 "미검출 0"에 대한 QA 지적(다중 끊김 67% 미검출)이 코드에서 근본 원인까지 수정·회귀검증 완료되었다**는 점이 강점이다. 이번 PDCA Act 1회차에서는 자동수정 가능 범위로 한정한 QA §2-C(최소 결함 길이 필터 근거)와 Gap G5(휨/끊김 분류 정교화) 2건을 추가로 대응했다(§3-1 참고). 다만 이는 여전히 **완전 합성 데이터** 위에서의 검증이며, spec §4가 요구한 **이상탐지 모델(PatchCore/PaDiM)** 접근과는 다른 고전적 정합+diff 방식이다. 실 CAD·목형 데이터 확보 후 재검증과 모델 접근 정합이 남은 최대 과제이며, 이는 이번 자동 개선 범위 밖(별도 데이터 확보/모델링 프로젝트 필요)이다.

## 3. QA 리포트와 현재 코드의 상태 차이 (중요)

과제①과 대비되는 강점: 과제②는 QA 지적 중 **2건(§2-A, §2-B)이 근본 원인 수정 + 회귀검증까지 코드에 반영 완료**되어 있다(README 버그수정 이력).

| QA 발견 | 대응(README 기록) | 재검증 상태 |
|---|---|---|
| §2-A 다중 동일결함 렌더링 소실(끊김3개 중 1개만 검출, 67% 미검출) | `Arc(points, orig_idx, is_closed)` 구조 도입 — 랩어라운드 병합을 최초 컷에만 적용, `start_frac` 기준점을 원본 둘레로 통일 | **완료** — case4~6 신규 회귀테스트 편입, 3/3 검출 확인 |
| §2-B ECC 180° 대칭 오수렴(45°에서 17건 오검출)인데 `converged=True` 반환 | 잔차(chamfer) 기반 `registration_reliable` 가드레일 추가(임계 10px=2.0mm), 결과 억제 없이 신뢰불가만 노출 | **완료** — 실측표(1.4°~90°)로 임계 근거 제시, case0~6 회귀 정상 |
| §2-C 최소 결함길이 2.0mm 필터 근거 | `sweep_min_defect_arc_threshold()` 스윕 테스트 추가 — 0.5~10.0mm 후보별 과검출/미검출 실측, 값은 2.0mm 유지 | **완료(PDCA Act 1회차)** — 실제 결정 요인이 `DEFECT_TOL_MM`/라인두께 기하학임을 확인, README에 실측표·해석 기록. 단 노이즈 없는 합성 데이터 한정 |
| §3 실데이터 확보 후 테스트 | 미대응 | **미완** — G1과 동일, 이번 자동 개선 범위 밖 |

→ 과제①의 최우선 Gap(G1: 치명 미검출0 재검증 미완)이 과제②에서는 이미 해소된 상태다. 다만 이 모든 검증이 **합성 데이터 한정**이라는 점은 G1(실데이터)으로 여전히 남는다.

### 3-1. 이번 PDCA Act 1회차에서 대응한 항목 (자동수정 가능 범위로 한정)

Gap 목록 중 G1(실데이터 확보)·G2(이상탐지 모델 도입)·G6~G10(검사자 UI·버전게이트·HITL·MES)은
코드 자동수정으로 해결 불가능한 대규모 신규 기능/데이터 확보 항목이므로 **이번 자동 개선 범위
밖**으로 명시하고 손대지 않았다(별도 프로젝트/데이터 확보 필요). 이번 사이클에서는 자동수정이
현실적으로 가능한 다음 2건만 대응했다.

1. **QA §2-C** — 최소 결함 길이 필터(`MIN_DEFECT_ARC_MM = 2.0mm`) 근거 미확인: 값 자체는
   바꾸지 않고, 1.0mm~3.0mm 등 후보 임계값 스윕 테스트(`sweep_min_defect_arc_threshold()`)를
   코드로 추가해 실측 근거를 README에 남겼다. 결과: 이 범위 내에서는 임계값이 검출 결과에
   영향을 주지 않으며(진짜 결정 요인은 `DEFECT_TOL_MM`/라인두께 기하학), 필터가 실제로
   결함을 놓치기 시작하는 지점은 4~5mm 이상부터라는 것을 확인 — 2.0mm는 안전거리가 충분한
   값임을 실측으로 뒷받침했다.
2. **Gap G5** — 휨/끊김 최종 분류 규칙 미정교화: 같은 위치에서 겹치는 휨/끊김을
   `resolve_bend_break_overlap()` 후처리로 `"복합(휨+끊김 의심)"`으로 명시 태깅하도록
   `die_blade_qc_demo.py`에 추가했다. 기존 case0~6 회귀 테스트는 전부 PASS로 유지됨을
   재실행으로 확인했다(자체 검증 로직도 복합 태그를 인식하도록 함께 갱신).

## 5. 정합률 산출 근거

- 전체 설계 범위: 핵심 축 15항목 가중 — 검출클래스(휨·끊김 충족2 / 위치오차·마모 미충족2), 정합(충족1), 합성결함(충족1), mm출력·세그(부분2), 이상탐지모델(미충족1), UI 화면①~⑤(미충족5), 이중검수(미충족1), 분류 정교화(G5, 충족0.5 신규 반영). 부분=0.5 가중 시 (충족4 + 부분2.5×0.5)/약15 ≈ **23%**. 이번 사이클은 축 개수를 늘리지 않고 기존 축(G5)의 완성도만 보강했으므로 전체 설계 범위 기준 변화는 미미하다.
- 프로토타입 선언 범위(휨·끊김 정합+diff 원리검증): 정합 수렴·합성결함 주입·case0~6 미검출0·다중결함(§2-A)·ECC가드레일(§2-B)·최소길이필터 근거(§2-C)·분류정교화(G5) = 대부분 충족. §3(실데이터 확보 후 테스트) 미완 감점 → 약 **93%**.
- 남은 감점 요인은 전적으로 "완전 합성 데이터"라는 한계(G1, §3)이며, 이는 코드 수정이 아니라 실측 데이터 확보가 있어야 해소되는 항목이라 이번 자동 개선 범위 밖으로 유지했다. 억지로 90%대 중반 이상을 맞추기 위해 이 항목을 임의로 "충족" 처리하지 않았다.
