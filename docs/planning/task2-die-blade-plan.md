# 기획서 — 과제② 목형(다이커팅 칼날) 불량검사 AI

> 근거 문서: `intro.md`(과제 정의), `problem.md`(문제·실패사례), `spec.md`(검출 클래스/데이터전략/지표), `prompt.md`(목표·제약).
> 본 문서와 상충되는 가정은 만들지 않았으며, 명시가 필요한 부분은 "가정:" / "확인 필요:"로 표기했다.

## 0. 과제 요약 (근거: intro.md §과제②, spec.md §1)

목형(다이커팅 칼날) 이미지를 설계 CAD/도면과 정합(Registration)한 뒤, 형상 차이를 이상탐지·세그멘테이션으로 검출한다. 검출 클래스는 정상/휨/끊김(최우선 치명)/위치오차/마모(연속량)이며, 최종 목표는 "끊김·휨 미검출 0"이다 (spec.md §2, §6).

## 1. 기능 요구사항 (우선순위 포함)

우선순위는 MoSCoW(Must/Should/Could/Won't-now)로 표기. 치명 클래스(끊김·휨) 관련 기능을 최상위에 배치한 것은 problem.md·prompt.md의 "미검출 0 최우선" 원칙에 따른 것이다.

## 2. 검사자 사용 시나리오

**가정:** 검사자는 비개발자이며 현재 목형을 자로 재거나 육안 대조하는 인력이라고 전제한다 (problem.md §1 근거).

### 시나리오 A — 정기 검사(신규 제작 목형 입고 검수)
1. 검사자가 고정 지그에 목형을 거치하고 촬영(또는 라인 카메라 자동 촬영).
2. 시스템이 자동으로 해당 제품코드의 CAD 도면을 조회해 오버레이 정합.
3. 화면에 원본 도면(초록) vs 실물 윤곽(빨강) 오버레이와 함께, 이탈 구간을 강조 표시.
4. 끊김/휨 의심 구간은 "치명" 배지로 최상단에 표시, 위치오차·마모는 보조 정보로 하단 표시.
5. 검사자가 강조 구간을 눈으로 재확인 후 승인/반려 클릭 → 결과가 목형ID·타발수량과 함께 MES에 자동 기록.

### 시나리오 B — 치명 결함 알림(끊김·휨 의심)
1. 모델이 끊김 의심 신뢰도 임계치 초과 감지 시 즉시 알림(붉은 경고) 발생.
2. 미검출 0 목표에 따라 임계치는 보수적으로(재현율 우선) 설정되어 있어 과검출이 다소 있을 수 있음 — 검사자가 재검 프로세스로 최종 확인.
3. 검사자가 확대 뷰에서 mm 단위 이탈 수치 확인 후 실물 재점검, 최종 판정은 사람이 확정 (prompt.md 한계: AI 단독 확정 금지).

### 시나리오 C — 마모도 추적(반복 사용 목형)
1. 동일 목형ID가 반복 촬영될 때마다 마모도 등급(정상/주의/교체) 또는 회귀 수치가 갱신되어 이력에 누적.
2. "주의" 등급 도달 시 교체 예정 알림, "교체" 등급 도달 시 즉시 사용 중단 권고.
3. 검사자는 수치 추세(그래프)를 보고 교체 계획을 수립 — 육안 대비 재현성 있는 정량 근거 확보.

### 시나리오 D — 도면 버전 불일치 예외 처리
1. 오검출이 반복 발생하는 목형에 대해 시스템이 "CAD 버전-생산 버전 일치 확인" 체크리스트를 우선 제시.
2. 버전 불일치 확인 시 모델 재검사 없이 데이터 관리팀에 반려 — 모델 결함으로 오인하지 않도록 함 (prompt.md 한계 근거).

## 3. 성공 기준 (spec.md §6 목표 지표 반영)

| 구분 | 지표 | 목표 | 근거 |
|---|---|---|---|
| 최우선 | 결함 재현율(끊김·휨) | 미검출 0 (Recall = 100% 지향, test셋 기준) | spec.md §6 |
| 보조 | IoU | 파일럿 단계 목표치 확인 필요 (수치 미정) | spec.md §6 |
| 보조 | mAP | 파일럿 단계 목표치 확인 필요 (수치 미정) | spec.md §6 |
| 보조 | 위치오차(mm) | 캘리브레이션 오차 허용범위 내 (지그·렌즈 사양에 따름, 확인 필요) | spec.md §1, §6 |
| 운영 원칙 | 오검출(과검출) | 재검 프로세스로 흡수 가능 — 별도 상한 지표 없음, 다만 재검 부담이 과도해지면 안 됨(상한 확인 필요) | spec.md §6, prompt.md 목표1 |
| 성숙도 | 사업계획 성숙도 단계 | "기초 → 중간1" 수준 달성 (완전 무인화 아님) | spec.md §6, prompt.md Non-goals |
| 데이터 전제 | 촬영 표준화 | 지그·조명·해상도 고정 완료 후에만 성능 주장 | prompt.md 한계, spec.md §7 |
| 데이터 전제 | 실불량 검증 | 합성 데이터로 학습한 모델은 반드시 실불량 test셋으로 별도 재검증, test셋 누수·소량 착시 경계 | prompt.md 한계 |
| 추적성 | MES 연계 | 모든 판정 결과가 목형ID·제품코드로 MES 조회 가능 | spec.md §7 |

**확인 필요:** IoU/mAP/위치오차(mm)의 구체적 목표 수치, 실불량 최소 확보 수량 및 확보 일정, 과검출 허용 상한, 엣지 vs 서버 추론 여부는 spec.md §8에 "미해결·후속 검토 사항"으로 명시되어 있으므로 본 기획서 수준에서 임의로 확정하지 않았다. 파일럿 착수 전 사업 주관부서·현업 검사팀과 확정 필요.

## 4. 단계별 일정 초안

**가정:** 각 단계의 기간(주 단위)은 spec.md/problem.md에 명시된 데이터 난제(실불량 극희소)의 규모를 고려한 통상적 추정이며, 실제 기간은 목형 형상 다품종성·CAD 정비 상태에 따라 조정 필요("확인 필요").

### 단계 1 — 데이터 확보 (예상 6~8주)
- 정상 목형 촬영: 형상·제품·크기별 다량 확보, 고정 지그·백라이트/텔레센트릭 렌즈 표준화 선행 (spec.md §3.1, §7)
- CAD/도면 기준 형상 등록 및 생산 버전 일치 검증 절차 수립 (spec.md §7)
- 목형ID·제품·검사결과 등 메타데이터 스키마 확정, MES 연계 항목 합의
- 폐기 목형 재활용 + 인위적 손상 샘플 제작 착수(끊김·휨 위주, 극희소 전제) (spec.md §3.1)
- 목형ID·제품 단위 train/val/test 분할 설계(누수 방지, 실불량은 test 중심 배치, 예시 70/15/15) (spec.md §3.3)
- Exit 조건: 정상 데이터 목표량 확보 + 촬영 표준화 완료 + CAD 오버레이 정합 파이프라인 1차 검증 완료

### 단계 2 — 프로토타입 (예상 6주)
- 정상 데이터 기반 이상탐지 모델(PatchCore/PaDiM 등) 1차 학습 (spec.md §4)
- 합성 결함 증강(국소 끊김·휨·마모·위치이동, CutPaste/CutMix) 파이프라인 구축 (spec.md §3.2)
- 정합(Registration) 정밀도 검증 — 정렬 오차로 인한 오검출 최소화가 선행 조건 (prompt.md 목표3)
- 소량 실불량 + 합성 데이터로 초기 성능 측정(끊김·휨 재현율 우선 확인)
- 판정 결과 시각화 프로토타입(도면 오버레이 강조 표시) 제작, 비개발자 검사자 피드백 1회 청취
- Exit 조건: 치명 클래스(끊김·휨) 재현율이 내부 목표선 이상 확인 + 오검출 수준이 재검 프로세스로 흡수 가능한 범위임을 확인

### 단계 3 — 파일럿 (예상 8~10주)
- 실제 현장 검사 라인 일부에 시범 적용(사람 검사와 병행·이중 확인)
- 실불량 test셋으로 리얼-합성 갭 검증(spec.md §8) — 합성만으로 성능 확정하지 않음 (prompt.md 한계)
- 마모도 등급화/회귀 모델 검증, MES 연계 실데이터 연동 테스트
- 검사자 UX 피드백 반영(시각화·알림 임계치 조정)
- CAD 버전 불일치 예외 처리 프로세스 실운영 검증
- Exit 조건: 목표 재현율(끊김·휨 미검출 0) 파일럿 라인에서 재현 확인 + 현업 검사자 승인 + MES 연계 정상 동작

### 단계 4 — 배포 (예상 4주 이후 지속 운영)
- 대상 라인 확대 적용, 운영 매뉴얼·재검 프로세스 문서화
- 지속적 리얼-합성 갭 모니터링 체계 정례화(spec.md §8)
- 목형 형상 다품종 확장 시 온보딩 절차 정립(F14 연계)
- 사람 재검 병행 체제 유지(완전 무인화 아님, prompt.md Non-goals) — 목표 성숙도 "기초→중간1" 달성 확인
- 확인 필요: 배포 규모(전체 라인 vs 일부 라인), 엣지/서버 추론 방식 최종 결정(spec.md §8 미해결)

## 5. 리스크 및 전제 (요약)

- 실불량 데이터가 태생적으로 희소하므로, 모든 단계에서 "소량 데이터 과대평가" 위험을 경계해야 한다 (prompt.md 한계).
- 촬영 표준화가 데이터 수집보다 선행되어야 하며, 미표준화 상태의 데이터로 도출한 성능은 신뢰할 수 없다.
- 마모는 이진 분류가 아닌 연속량/등급으로 다뤄야 하므로, 성공 기준·라벨링·UI 모두 이를 반영해야 한다.
- 최종 출하 판정은 항상 사람이 확정하며, AI는 검출·표시까지만 수행한다(법적·안전상 리스크가 큰 치명 결함일수록 이 원칙이 중요).
