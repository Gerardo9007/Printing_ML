# 과제① 인쇄판 문안검사 — 검사자용 결과 확인 UI/UX 설계

## 0. 근거 문서 및 설계 전제

본 설계는 `intro.md`, `problem.md`, `spec.md`, `prompt.md`에 근거한다. 상충되는 임의 가정은 두지 않았으며, 문서에 없는 부분은 아래와 같이 "가정:"으로 명시한다.

- **재현율(미검출 방지) 최우선** (`prompt.md` 목표1, `spec.md` §6): 화면 설계는 "치명 클래스(성분표시오류)는 AI가 최종 확정하지 않고 반드시 사람 재검으로 귀결"되도록 강제하는 흐름을 포함해야 한다 (`prompt.md` 한계: "법적/규제 리스크가 있는 클래스는 최종 판단을 AI가 단독으로 확정하지 않는다").
- **비개발자 검사자가 이해 가능한 설명** (`prompt.md` 목표5): 전문 용어 없이 "정답과 무엇이 다른지"를 시각적으로 보여줘야 한다.
- **출력 스키마** (`spec.md` §5): 불량 종류 + BBox/영역 좌표 + 텍스트 diff. UI의 결과 표시 항목은 이 스키마를 그대로 반영한다.
- **정합(Registration) 전제** (`spec.md` §3.3, `prompt.md` 목표3): 정렬 오차가 오검출의 주요 원인이므로, 화면에는 정합 상태(정합 성공/실패, 정합 신뢰도)를 노출해야 한다.
- **버전 관리** (`spec.md` §7): 디자인 원본 버전과 실제 생산 버전 불일치가 오검출 주요 원인 → 업로드 단계에서 버전 검증 게이트를 둔다 (기획 문서 `docs/planning/task1-print-plate-plan.md`의 P0 "정합+버전검증게이트"와 일치).
- **MES 연계 메타데이터** (`spec.md` §3.1): 제품코드·LOT·인쇄판버전·원본링크·검사결과·불량유형·좌표를 화면에 노출/기록한다.
- **mm 단위 오차 표시**: `spec.md` §1 표에서 mm 단위 위치오차는 과제②(목형)의 출력 항목으로 명시되어 있다. 과제①의 출력 항목(§5)에는 "텍스트 diff"만 있고 mm 단위 오차는 명시되어 있지 않다.
  - **가정**: 다만 사용자 요청에 "②mm 단위 오차 표시"가 과제① UI 요건으로 명시되었고, 실제로 인쇄 위치 편차(예: 문구가 기준 위치에서 밀림, 성분표시 박스 위치 이동)는 인쇄 현장에서 실무적으로 의미 있는 정보이므로, 본 설계는 **정합 후 산출되는 픽셀 오차를 스캔 DPI 기준으로 mm 환산하여 보조 정보로 표시**하는 것으로 가정한다. 이는 §1 정합(Registration) 단계의 부산물로 자연스럽게 얻어지는 값이며 spec.md와 상충되지 않는다. 실제 도입 시 정밀도 검증은 "확인 필요".
- **엣지/서버 추론 여부, 처리 지연 시간**: `spec.md` §8 미해결 사항. UI는 두 경우 모두 대응 가능하도록 "처리 중" 상태를 명시적으로 설계하고, 구체적 SLA 수치는 표기하지 않는다.

## 1. 사용자 및 사용 맥락

- **주 사용자**: 인쇄 현장 검사자 (비개발자, 도메인 전문가). 하루 다수 LOT을 반복 검사.
- **기기 가정**: **가정**: 검사대 옆 태블릿 또는 데스크톱 모니터 사용을 전제로 한다 (스캐너/카메라 지그와 연결된 고정 워크스테이션). 모바일 전용 UI는 범위에서 제외.
- **핵심 원칙**: AI는 "치명 결함일 가능성이 있는 곳"을 놓치지 않고 보여주는 보조자이며, 최종 출하 판정은 항상 사람이 내린다 (`prompt.md` 한계 조항).

## 2. 전체 화면 흐름 (Flow)

```
┌──────────────┐     ┌──────────────┐     ┌──────────────────┐     ┌──────────────┐     ┌──────────────┐
│ ① LOT/대상   │ --> │ ② 촬영·업로드│ --> │ ③ AI 처리 대기    │ --> │ ④ 판정 결과   │ --> │ ⑤ 재검 처리   │
│   정보 입력   │     │   & 정합 확인 │     │   (정합→OCR→Diff) │     │   확인 화면    │     │   & 완료 기록  │
└──────────────┘     └──────────────┘     └──────────────────┘     └──────────────┘     └──────────────┘
                                                                          │
                                                                          ├─ 치명 클래스 검출 시
                                                                          │   → 강제 재검 라우팅 (건너뛰기 불가)
                                                                          └─ 비치명/정상
                                                                              → 검사자 확인 후 종결 가능
```

### 2.1 단계별 설명

**① LOT/대상 정보 입력**
검사자가 제품코드·LOT·인쇄판버전을 선택 또는 스캔(바코드)으로 입력. 시스템은 이 시점에 MES에서 해당 LOT에 매칭된 디자인 원본과 원본 버전을 자동 조회한다.

**② 촬영·업로드 & 정합 확인**
- 스캐너/라인스캔카메라 지그로 촬영하거나, 사전 촬영된 이미지를 업로드.
- **버전 검증 게이트**: 조회된 디자인 원본 버전과 입력된 인쇄판버전이 일치하는지 시스템이 먼저 확인한다. 불일치 시 AI 검사로 진행하지 않고 즉시 경고 화면으로 분기한다 (아래 4.4절).
- 정합(Registration) 자동 수행 후 정합 신뢰도를 % 또는 등급(성공/주의/실패)으로 표시. 정합 실패 시 재촬영을 유도한다.

**③ AI 처리 대기**
정합 → OCR → 픽셀 차분 파이프라인 진행 상태를 표시(단계별 체크리스트 형태). 엣지/서버 추론 여부에 따라 소요 시간이 다를 수 있으므로 절대 시간이 아닌 "진행 단계"로 표현.

**④ 판정 결과 확인 화면** (핵심 화면, 3장에서 상세 설명)

**⑤ 재검 처리 & 완료 기록**
검사자의 최종 의사결정(정상 출하 / 재검 필요 / 폐기)을 기록하고 MES로 결과를 전송. 치명 클래스가 하나라도 검출된 LOT은 "재검 필요" 또는 "폐기" 외의 선택지를 비활성화한다.

## 4. 불일치 시각화 방식 상세

### 4.1 ① 원본 대비 Diff 하이라이트

**목적**: 전문 용어 없이 "다른 곳이 여기다"를 직관적으로 보여준다 (`prompt.md` 목표5).

3가지 보기 모드를 토글로 제공한다.

```
[모드 1: 좌우 비교]           [모드 2: 겹쳐보기(오버레이)]      [모드 3: 하이라이트만]
┌───────┐  ┌───────┐         ┌─────────────────┐              ┌───────────────────┐
│ 원본  │  │인쇄물 │         │ 원본(회색 반투명) │              │  인쇄물 이미지       │
│       │  │       │         │  + 인쇄물(선명)   │              │  + 결함 영역만       │
│ 문안A │  │ 문안A'│         │  겹친 상태         │              │  박스/하이라이트     │
│(기준) │  │(실측) │         │  불일치=색 강조    │              │  표시                │
└───────┘  └───────┘         └─────────────────┘              └───────────────────┘
```

- **색상 규칙** (심각도 기반, `spec.md` §2 치명 클래스 정의 반영):
  - 🔴 빨강 실선 박스 = 치명 클래스(성분표시오류) — 반드시 눈에 띄는 최우선 색상, 리스트 최상단 고정
  - 🟡 노랑 점선 박스 = 비치명(오탈자, 점누락, 번짐)
  - 회색 음영 = 원본에는 있으나 인쇄물에서 사라진 요소(문자 누락) — "빠진 자리"를 보여줘 누락을 직관적으로 인지시킴
- 각 박스에 마우스오버/탭 시 "원본: OO / 인쇄물: XX" 텍스트 말풍선 표시 → OCR 근거를 사람이 바로 읽을 수 있게 함 (`spec.md` §7 설명 가능성 요구사항 반영).
- 확대/축소(핀치 줌 또는 +/- 버튼) 지원 — 미세 결함(점·자간)은 확대 없이는 육안 확인이 어려움 (`problem.md` "미세 결함 취약" 대응).

### 4.2 ② mm 단위 오차 표시

**가정 재확인**: `spec.md` §5에서 과제①의 라벨 항목은 "불량 종류 + BBox/영역 좌표 + 텍스트 diff"이며 mm 오차는 과제②의 항목으로 명시되어 있다. 아래는 정합 단계 부산물을 활용한 보조 표시 방법에 대한 가정 기반 설계이며, 실제 필요성/정밀도 기준은 "확인 필요".

```
   결함 #3 상세 (점누락)
   ┌─────────────────────────────┐
   │        기준 위치 ✕            │
   │          │                   │
   │          │ 0.3mm             │  ← 원본 대비 기준 위치에서
   │          ▼                   │     인쇄물 상 마침표가 사라진 지점까지의
   │        (없음 - 인쇄물)        │     거리를 정합 좌표계 기준 mm로 환산
   └─────────────────────────────┘
   [적용 조건] 정합 신뢰도가 일정 수준 이상일 때만 mm 값 노출.
   정합 신뢰도가 낮으면 "mm 오차: 산출불가(정합 재확인 필요)"로 표시.
```

- 표시 원칙: mm 값은 **텍스트/기호 결함의 "위치가 원본과 얼마나 벗어났는가"**를 보조 설명하는 값이며, 문자 누락·오탈자처럼 위치가 아닌 "내용"이 틀린 결함에는 "위치오차: 해당없음(텍스트결함)"으로 명확히 구분 표시한다. 이는 오차 수치가 없다고 결함이 경미하다는 오인을 막기 위함.
- mm 환산은 스캔 해상도(DPI, `spec.md` §3.1 "600dpi 이상 스캔")를 기준으로 계산하며, 촬영 표준화가 선행되지 않은 경우(비표준 촬영본) mm 값 표시 자체를 비활성화하고 "촬영 표준화 미확인 — mm 오차 미제공"으로 표기한다 (`prompt.md` 한계: "촬영 표준화가 선행되지 않은 데이터셋에 대한 성능 주장은 하지 말 것" 원칙을 UI에도 준용).

### 4.3 결함 심각도 요약 카드 (비개발자용 한 줄 요약)

리스트 최상단에 아래와 같은 신호등 요약 카드를 배치해, 세부 화면을 열지 않아도 판정 성격을 즉시 파악하게 한다.

```
┌───────────────────────────────────────────┐

### 4.4 버전 불일치 경고 화면 (별도 분기)

## 5. 재검 처리 화면 (⑤)

```
┌─────────────────────────────────────────────────────────────┐

│  ─────────────────────────────────────────────────────────  │

│                                                               │
│  검사자 코멘트: [____________________________]                │
│                                                               │
│  [제출 → MES 기록]                                            │

- 제출 시 불량 유형·좌표·mm오차(해당 시)·검사자 최종 판정·코멘트가 MES로 함께 전송된다 (`spec.md` §3.1 메타데이터, §7 추적성 요구사항).

## 7. 확인 필요 (미해결 사항)

- mm 단위 오차 표시의 실제 도입 필요성 및 정밀도 기준 — `spec.md` §8에 실불량 최소 수량·엣지/서버 여부 등은 미해결로 명시되어 있으며, mm 오차 표시 자체도 과제①에 원래 명시된 항목이 아니므로 현업 검사자 인터뷰를 통한 요구 재확인이 필요.
- 태블릿/데스크톱 등 실제 사용 기기 형태 — 가정으로 처리, 확정 필요.
- 정합 신뢰도의 구체적 임계값(몇 % 이하면 재촬영 유도할지) — 미정.
