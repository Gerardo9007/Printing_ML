## 0. 공통 추정 방법론

실불량 최소 수량을 정할 때, 두 과제 모두 목표가 "치명 클래스 미검출 0"(spec.md §6, prompt.md 목표1)이므로 단순 "많이 모은다"가 아니라 **"미검출률이 특정 상한 이하라고 통계적으로 주장할 수 있는 최소 검증 표본 수"**를 기준으로 잡는다.

- Rule of Three: 표본 n건 중 결함이 0건 관찰되면, 실제 결함(미검출)율의 95% 신뢰상한은 근사적으로 `3/n`이다.
- 예: 미검출률 상한을 1%로 주장하려면 `n ≥ 300`, 상한 3%면 `n ≥ 100`.
- 치명 클래스(성분표시오류·오탈자 / 칼날 끊김)는 test셋에 최소 이 규모가 있어야 "미검출 0에 가깝다"는 주장이 통계적으로 의미를 갖는다. 실불량만으로 이 규모를 못 채우는 것이 두 과제의 근본 난제(problem.md §2)이므로, 부족분은 **인위적으로 재현한 실물 손상 + 검증된 합성 표본**으로 보완한다는 전제를 깐다.
- 가정: 이 300건 기준은 신뢰수준 95%·상한 1%를 목표로 잡은 것이며, 사업계획상 목표 성숙도가 "기초→중간1"(spec.md §6)임을 고려하면 1단계 목표는 상한 3~5%(n=60~100)로 낮춰 잡는 것도 현실적 대안이다. 최종 기준선은 사업 요구사항 확정 후 조정 필요 — **확인 필요**.

## 1. 과제① 인쇄판 문안검사

### 1.1 최소 데이터 수량 추정

| 구성 | 추정치 | 근거 |
|---|---|---|
| 정상 정합쌍(원본↔인쇄물) | 가정: SKU 50종 × 언어/폰트 변형 3종 = 150조합 × 30쌍 ≈ **4,500쌍** | 제품·폰트·언어별 폭넓은 분포 확보(spec.md §3.1) |
| 합성 불량(클래스당) | 문자누락·오탈자·점누락·번짐 각 2,000~3,000장 | 딥러닝 소규모 파인튜닝 시 클래스당 수백~수천 장이 일반적 하한 |
| 합성 불량(성분표시오류) | **3,000~5,000장 (타 클래스의 1.5~2배)** | 치명 클래스(법적 리스크) → 검출력 확보 위해 비중 상향(spec.md §3.3 클래스 불균형 보정) |
| 실불량(클래스당, 학습+검증 합산) | 목표 20~30건/클래스 (6개월~1년 누적) | 반려품 자체가 극희소(problem.md §2) — "최대한 모으는 것"이 목표치이지 통계적 충분치는 아님 |
| 치명 클래스 test 표본(실불량+인위재현 합산) | **≥300건** (95% 신뢰, 미검출 상한 1% 기준) 또는 완화 시 ≥100건 (상한 3%) | 위 §0 Rule of Three |
| 학습용 총 규모(가정) | 정상 4,500 + 합성 5클래스 평균 2,600×5 ≈ 13,000 + 실불량 150 ≈ **약 17,000~18,000장** | 상기 합산 |

확인 필요: 실제 SKU 수·언어/폰트 변형 수는 사내 제품 마스터 데이터로 검증해야 하며, 위 조합수(150)는 가정치다.

### 1.2 합성 증강 파이프라인 (구체 절차)

**라이브러리/도구**
- PDF/AI 벡터 라스터화: `PyMuPDF(fitz)` 또는 `pdf2image`(600dpi 이상)
- 텍스트 좌표 추출: PDF 텍스트 오브젝트 파싱(`PyMuPDF`의 `get_text("dict")`)으로 문자 단위 bbox 확보
- 이미지 처리: `OpenCV`, `Pillow`
- 증강 파라미터화: `Albumentations` (밝기/대비/채도/노이즈/미세회전)
- OCR 자동 검증: `PaddleOCR` 또는 `EasyOCR` (한글 포함 인식 지원)
- 참고 유사 오픈소스 기법: TextRecognitionDataGenerator(TRDG), SynthTIGER — 텍스트 결함 합성 방식 참조용

**절차 (순서대로)**
1. 디자인 원본(AI/PDF)을 600dpi 이상으로 라스터화 → GT 이미지, 동시에 문자 단위 bbox·문자열 메타 추출.
2. 원본 문자열에서 결함 유형별로 국소 변형을 프로그램 생성(모두 라벨 자동 부여):
   - **오탈자**: 혼동쌍 사전(예: O/0, l/1, ㅇ/0, 자모 유사쌍) 기반 랜덤 치환 → 해당 bbox 영역만 동일 폰트/크기로 재렌더링.
   - **문자 누락**: 문자열에서 1글자 삭제 후 해당 영역을 배경색으로 인페인팅(주변 색상 샘플링), 필요 시 자간 유지.
   - **점·기호 누락**: 정규식으로 마침표/쉼표/®·™ 위치 탐지 후 픽셀 단위로 제거.
   - **번짐**: `cv2.GaussianBlur`(σ=0.5~1.5) + 모폴로지 팽창(커널 2~3px)을 국소 마스크에 적용해 잉크 확산 재현.
   - **성분표시오류**: 성분표 텍스트 블록 내에서 위 문자누락/오탈자 결합 + 항목 순서 변경(위치오류) 케이스 추가. 치명 클래스이므로 생성 비율을 상향(§1.1).
3. 공통 후처리 지터 적용(Albumentations): 밝기/대비/채도 ±10~20%, Gaussian/salt-and-pepper 노이즈, JPEG 압축 아티팩트, 미세 회전 ±1~2°(spec.md 제약: 과회전·반전 금지, ①에서 좌우/상하 반전은 사용 안 함).
4. 생성 시점에 결함 유형·bbox·정답 텍스트 대비 diff를 JSON/COCO 형식으로 동시 기록(별도 수동 라벨링 불필요).
5. **자동 라벨 self-check**: 생성 이미지에 OCR 재적용 → 의도한 결함이 OCR 결과에도 반영되는지 확인(예: 삭제 문자가 인식 결과에서도 빠졌는지). 렌더링 깨짐·예상치 못한 결과는 폐기.
6. 클래스별 목표 비율에 맞춰 생성량 재조정(성분표시오류 상향, spec.md §3.3 클래스 불균형 보정과 연결).
7. 실측 정상 샘플에서 노이즈 통계(분산, 색 편차 범위)를 추출해 합성 데이터의 지터 파라미터 범위를 보정 → 리얼-합성 갭 축소(1차 보정, 최종 검증은 §1.4 체크리스트).

### 1.3 실불량 데이터 확보 계획

- **반려품 아카이빙 프로세스 신설**: 현재는 검사자가 반려 판정 후 바로 재작업/폐기로 넘어가 기록이 사라짐(problem.md 공통 문제 "기록 부재"). 반려 확정 시 촬영 후 보관하는 단계를 공정에 추가.
- **소급 수집**: 과거 반려 LOT의 사진 기록이 남아있는지 확인(확인 필요: 기존 품질부서 기록 유무).
- **인위적 실물 결함 재현**: 실제 인쇄 공정에서 의도적으로 세팅을 어긋내어(예: 저해상도, 특정 문자 판 손상) 오탈자·번짐 실물을 소량 재현 — 순수 합성과 달리 "실제 인쇄 물리 특성"을 담을 수 있는 유일한 방법. 성분표시오류처럼 발생 빈도 자체가 극히 낮은 클래스는 이 방법에 크게 의존.
- **목표치**: 클래스당 20~30건을 6개월~1년 내 누적, 치명 클래스(성분표시오류)는 인위적 재현으로 목표치 우선 확보.
- **MES 연계**: spec.md §3.1 메타데이터 스키마(제품코드·LOT·인쇄판버전·검사결과·불량유형·좌표)를 그대로 반려품 아카이빙에 적용해 향후 수집을 자동화.

### 1.4 데이터 품질 검증 체크리스트

- [ ] 원본-인쇄물 정합쌍의 LOT·버전 일치 확인 (버전 불일치는 데이터 관리 문제, 모델 성능 이슈로 오인하지 않음 — prompt.md 제약)
- [ ] 해상도 600dpi 이상 충족
- [ ] 촬영 조건(조명·ICC 색프로파일) 기록 존재
- [ ] 합성 결함 라벨 좌표 ↔ 실제 픽셀 위치 일치 여부 샘플 검수(예: 매 배치 5% 샘플링)
- [ ] 클래스별 샘플 수·비율 기록 및 목표 비율과의 편차 확인
- [ ] 2인 교차검수 일치율(Cohen's kappa 등) 산출 및 기준치(예: κ≥0.8) 충족 확인 — 기준치는 가정, 확인 필요

- [ ] 치명 클래스 test 표본 수 ≥ §1.1 산정 하한(최소 100, 목표 300) 충족

- [ ] 혼동쌍 사전 최신화 여부(신규 폰트/언어 추가 시 갱신)

## 2. 과제② 목형 칼날검사

### 2.1 최소 데이터 수량 추정

이상탐지(PatchCore, PaDiM) 기반 접근(spec.md §4)은 "정상 데이터 다량"이 핵심이므로, 정상 확보량이 결함 라벨 수량보다 더 중요하다.

| 구성 | 추정치 | 근거 |
|---|---|---|
| 정상 목형 | 가정: 형상 그룹 15~20종 × 150~200장 = **2,250~4,000장** | 이상탐지 벤치마크(예: MVTec AD류)에서 클래스당 정상 200장 내외가 baseline 가능한 규모라는 통상적 경험치 — 가정, 확인 필요 |
| 합성 결함(유형당) | 휨·끊김·위치오차 각 1,500~3,000장 | 합성 결함 삽입이 최대 효과 기법(spec.md §3.2) |
| 마모(연속량) 샘플 | 사용수명 구간(0/25/50/75/100%) × 최소 20장 = **100+장** | 마모는 회귀/등급 문제이므로 구간별 표본 필요(spec.md §4, prompt.md 제약: 이진분류 부적합) |
| 실불량(형상그룹×결함유형) | 최소 5~10건/조합, 누적 100건 이상 | 폐기목형 소실 구조적 제약(problem.md §2) — 현실적 목표치 |
| 끊김(치명 클래스) test 표본(실물+인위재현 합산) | **≥300건** (95%·상한1%) 또는 완화 시 ≥100건(상한3%) | §0 Rule of Three, 실물만으로 불가하므로 인위적 손상 재현으로 보완 |
| 학습용 총 규모(가정) | 정상 3,000 + 합성 4유형 평균 2,000×4=8,000 + 실불량 100~150 ≈ **약 11,000~11,500장** | 상기 합산 |

확인 필요: 형상 그룹 수(15~20종)는 실제 목형 품목 마스터로 검증 필요.

### 2.2 합성 증강 파이프라인 (구체 절차)

**라이브러리/도구**
- CAD/도면 파싱: `ezdxf`(DXF) 또는 `svgpathtools`(SVG 도면인 경우)
- 형상 처리: `OpenCV`(Canny 에지, 모폴로지), `scikit-image`(warp/elastic transform), `shapely`(폴리라인 연산)
- 증강: `Albumentations`(마스크 동기화 지원)
- 스케일 캘리브레이션: 체커보드 기반 `OpenCV camera calibration` 루틴

**절차 (순서대로)**
1. CAD/도면(DXF/SVG)을 파싱해 기준 형상 폴리라인 좌표 추출 → 스케일 캘리브레이션(체커보드·캘리브레이션 타겟)으로 px↔mm 변환 계수를 적용해 이미지 좌표계의 GT 마스크 생성.
2. 정상 목형 실측 이미지에서 Canny 에지 추출 → 기준 형상과 호모그래피/ICP로 정합(오버레이 정렬).
3. 결함 유형별 주입(마스크/에지 레벨에서 직접 조작 후 렌더링, 라벨 자동 부여):
   - **끊김**: 에지 라인의 랜덤 구간(길이는 mm 단위로 지정 후 캘리브레이션 계수로 픽셀 환산, 예: 2~10mm)을 배경색으로 삭제. 절단 형태(직선/톱니) 랜덤화.
   - **휨**: 국소 구간에 `scikit-image.transform.warp`(변위량을 mm로 지정) 적용, 실패 사례에서 관찰된 "기준선 바깥쪽으로 굽음" 방향성을 우선 재현(problem.md §1 실패 사례).
   - **위치오차**: 전체 또는 부분 형상을 수 mm 평행이동, 코너/기준점 위주로 이동량 부여.
   - **마모**: 에지에 Gaussian blur + 모폴로지 erode/dilate 강도를 0~100% 연속값으로 램프 적용, 그 연속값을 회귀 라벨로 동시 기록.
4. **CutPaste/CutMix**: 정상 이미지 내 임의 패치를 다른 위치(또는 다른 정상 이미지)에서 잘라 붙여 국소 이상 생성 — 라벨없는 이상탐지 학습 보강용(spec.md §3.2, ②에 특히 효과적).
5. 기하 증강: 0~360° 회전(형상 검사이므로 가능, spec.md §3.2) — 회전 시 GT 마스크도 동일 변환으로 동기화. 좌우/상하 대칭 플립은 형상이 대칭인 품목에만 적용(품목별 대칭 여부 메타데이터 필요, 확인 필요).
6. 조명/반사 시뮬레이션: 방향성 조명을 가정한 단순 shading 합성 또는 여러 실측 조명 조건 촬영본을 재조합, 편광 촬영 조건 재현.
7. 배경·이물 노이즈 합성(먼지·그림자 등 방해요소 삽입, 오검출 저감 목적).
8. 라벨 기록: 결함 유형 + 세그멘테이션 마스크 + 위치오차(mm, 캘리브레이션 계수로 환산) + 마모도(연속값)를 동시 저장.
9. **역산 검증**: 생성된 합성 이상 이미지에 Canny 재추출 후, 기준선과의 실측 편차(mm)가 의도한 주입값과 일치하는지 자동 검증 → 불일치 시 폐기.

### 2.3 실불량 데이터 확보 계획

- **폐기 목형 재활용**: 교체·폐기 예정 목형을 폐기 직전 표준 지그로 촬영 후 보관하는 절차를 신설(현재는 폐기와 동시에 데이터가 영구 소실 — problem.md §2 핵심 난제).
- **인위적 손상 샘플 제작**: 정상 목형에 통제된 절단(끊김)·벤딩(휨)·재조립(위치오차)을 실험적으로 가해 실물 손상 샘플을 확보 — 폐기목형 확보가 근본적으로 제한적이므로 이 방법이 유일한 규모 확대 수단.
- **마모 진행 시퀀스 구축**: 사용횟수·타발수량 로그가 있는 현장 목형을 사용수명 구간별(예: 25% 지점마다)로 주기 촬영해, 마모 회귀 라벨의 실측 ground truth 시퀀스를 확보.
- **목표치**: 형상그룹×결함유형 조합당 5~10건, 전체 누적 100건 이상(6개월~1년). 끊김(치명 클래스)은 인위적 손상 제작으로 §2.1의 하한(최소 100, 목표 300)에 우선 근접.
- **MES 연계**: 목형ID·누적 사용횟수·타발수량·검사결과·불량유형·불량위치 스키마(spec.md §3.1)를 그대로 적용.

### 2.4 데이터 품질 검증 체크리스트

- [ ] CAD/도면 버전과 실제 생산 목형 버전 일치 확인(버전 불일치는 모델 결함이 아님 — prompt.md 제약 우선 배제)
- [ ] 스케일 캘리브레이션(px↔mm) 정확성 및 최신성(재측정 주기, 가정: 분기 1회 — 확인 필요)
- [ ] 촬영 지그의 각도·거리·조명 재현성(고정 지그 이탈 여부)
- [ ] 반사·하이라이트로 인한 에지 왜곡 샘플 점검(편광/텔레센트릭 렌즈 효과 확인)
- [ ] 형상 정합(오버레이) 정렬오차가 허용치 이내인지 확인
- [ ] 마모 라벨(연속값/등급)과 실측 사용횟수의 상관성 검토(단조 증가 관계 확인)
- [ ] 이중검수 일치율 산출 및 기준치 충족
- [ ] 목형ID·제품 단위 train/val/test 누수 없음
- [ ] 끊김(치명 클래스) test 표본 수 ≥ §2.1 산정 하한(최소 100, 목표 300) 충족
- [ ] 리얼-합성 갭 정기 검증: 인위적 실물 손상의 에지 특성(블러 반경, 절단면 형태)과 합성 결함의 에지 특성을 통계적으로 비교

## 3. 가정 및 확인 필요 항목 요약

**가정 (문서 내 명시된 것 재정리)**
- 인쇄판 SKU 50종 × 언어/폰트 변형 3종 = 150조합, 목형 형상 그룹 15~20종.
- Rule of Three 기준 신뢰수준 95%·미검출 상한 1%(완화 시 3%)를 1단계 목표선으로 채택.
- 이상탐지 정상 데이터 클래스당 150~200장을 baseline 가능 규모로 채택(MVTec AD류 통상 경험치 기반).
- 마모 사용수명 구간을 0/25/50/75/100% 5단계로 나눔.
- 데이터 품질 체크리스트의 kappa 기준치(κ≥0.8), 캘리브레이션 재측정 주기(분기 1회)는 임의 제안치.

**확인 필요 (착수 전 검증 대상)**
- 실제 SKU/언어/폰트 조합 수, 목형 형상 그룹 수(제품 마스터 데이터 대조).
- 과거 반려품 사진 기록 존재 여부와 접근 가능성.
- 치명 클래스 미검출 상한을 1%로 할지 3%로 할지에 대한 사업적 의사결정(spec.md §8 미해결 항목과 연동).
- 목형 품목별 대칭 여부 메타데이터의 존재/구축 필요성.
- 리얼-합성 갭 검증 주기 및 통과 기준(정량 임계값)의 사업적 확정.
